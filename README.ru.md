# 2DTanks

2DTanks - браузерная многопользовательская игра про танки, разработанная в рамках дипломного проекта на JavaScript/TypeScript-стеке. Проект показывает реализацию сетевого взаимодействия в реальном времени, авторитетной серверной симуляции, обработки столкновений, хранения данных и React-клиента, который запускается прямо в браузере.

Основная задача проекта - поддерживать согласованное состояние общего физического игрового мира для нескольких игроков. Клиенты отправляют события ввода, сервер проверяет их, изменяет авторитетное состояние мира и рассылает подключенным игрокам синхронизированные снимки состояния через WebSocket.

## Возможности

- Регистрация, вход, JWT-аутентификация и редактирование профиля пользователя.
- Онлайн-комнаты для одиночной практики, многопользовательских матчей, наблюдателей, чата, приглашений и переподключений.
- Игровой процесс в реальном времени через WebSocket с серверным управлением комнатами и синхронизацией состояния.
- Настройка танка: корпус, гусеницы, башня, оружие, цвет, именованные пресеты и итоговые характеристики.
- 2D-движение, стрельба, разрушаемые объекты, предметы, обработка столкновений и результаты матча.
- Друзья, поиск пользователей, публичная галерея повторов, ссылки на повторы, лайки и воспроизведение матчей.
- Хранение в PostgreSQL пользователей, профилей, друзей, пресетов танков, матчей, событий повторов и статистики.
- Docker Compose-конфигурация для PostgreSQL, Node.js-сервера и фронтенда через Nginx.

## Архитектура

Приложение построено по клиент-серверной архитектуре с авторитетным игровым сервером.

- **Фронтенд**: одностраничное приложение на React и TypeScript. Отвечает за маршрутизацию, состояние авторизации, экраны интерфейса, ввод игрока, WebSocket-обмен, отрисовку через canvas/спрайты, звук и воспроизведение повторов.
- **Сервер**: приложение на Node.js и TypeScript. Express используется для REST API, а `ws` - для игровых WebSocket-соединений. Сервер хранит комнаты, сессии и состояние игроков, выполняет игровые тики, проверяет столкновения, рассылает снимки состояния, обрабатывает чат, отметки, приглашения, наблюдателей и переподключения.
- **База данных**: PostgreSQL хранит долгоживущие данные приложения: аккаунты, профили, друзей, пресеты танков, сведения о матчах, ввод и события повторов, лайки и статистику.
- **Docker-окружение**: Nginx раздает собранный фронтенд и проксирует `/api` и `/game` на сервер, поэтому браузер работает через один origin на порту `5173`.

## Модель данных

Схема базы данных построена вокруг пользователей и матчей. Пользователи связаны с профилями, друзьями и пресетами танков; матчи - с участниками, записанным вводом, метаданными повторов, событиями воспроизведения и данными публичной галереи.

## Быстрый запуск через Docker

Из корня репозитория:

```bash
docker compose up --build
```

Сервисы:

- Фронтенд: `http://localhost:5173`
- Серверный API и WebSocket напрямую: `http://localhost:3000`
- PostgreSQL: `localhost:5432`

Docker Compose создает базу PostgreSQL `tanks2d` с пользователем `tanks` и паролем `tanks`. Контейнер сервера применяет миграции при старте. Контейнер фронтенда раздает статические файлы через Nginx и проксирует API/WebSocket-запросы в контейнер сервера.

## Настройка сервера

```bash
cd server
npm ci
cp .env.example .env
npm run migrate
npm run dev
```

Перед запуском миграций без Docker задайте `DATABASE_URL` и `JWT_SECRET` в `server/.env`.

Полезные команды сервера:

```bash
npm run build
npm start
npm run dev:watch
npm run db:seed:demo
```

Важные переменные окружения:

- `PORT` или `SERVER_PORT`: порт HTTP и WebSocket, по умолчанию `3000`.
- `DATABASE_URL`: строка подключения к PostgreSQL.
- `JWT_SECRET`: длинная случайная строка для подписи токенов.
- `CORS_ORIGINS`: необязательный список дополнительных browser origin через запятую.

## Настройка фронтенда

Сначала запустите сервер, затем клиент:

```bash
cd frontend
npm ci
npm run serve
```

В режиме разработки webpack-dev-server слушает `http://localhost:8081` и проксирует `/api` на `http://127.0.0.1:3000`.

Команды сборки:

```bash
npm run build-dev
npm run build-prod
```

Production-сборка записывает браузерный bundle в `frontend/src/js/bundle.js`; Docker-образ затем раздает его через Nginx.

## Тестирование

Автоматизированные тесты реализованы для серверной части:

```bash
cd server
npm run test:unit
npm run test:integration
npm test
npm run test:coverage
```

Модульные тесты покрывают геометрию, обнаружение и разрешение столкновений, вспомогательную логику аутентификации, утилиты комнат, CORS, игровые константы, WebSocket-helpers и чистую логику репозиториев. Интеграционные тесты используют Supertest для Express-приложения; сценарии с базой данных требуют настроенной тестовой PostgreSQL-базы в `server/.env.test`.

В пояснительной записке также описаны ручные проверки интерфейса: вход, регистрация, навигация, игровые комнаты, настройка танка, игровой процесс, результаты матча, просмотр повторов, друзья и настройки.
