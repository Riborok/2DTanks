# Описание инфологической модели базы данных 2DTanks

## Введение

Инфологическая модель описывает предметную область многопользовательской онлайн-игры 2DTanks без привязки к конкретной СУБД. Модель нормализована, исключает дублирование данных и учитывает расширяемость системы (рейтинги, сезоны, реплеи, статистика).

---

## Функциональные области и сущности

### 1. Пользователи и профили

#### User (Пользователь)
**Назначение:** Учётная запись игрока в системе.

| Атрибут      | Описание                    | Ключ     |
|-------------|-----------------------------|----------|
| user_id     | Уникальный идентификатор    | PK       |
| login       | Логин для входа             |          |
| email       | Адрес электронной почты     |          |
| password_hash | Хеш пароля                |          |
| display_name | Отображаемое имя игрока    |          |
| created_at  | Дата и время регистрации    |          |
| updated_at  | Дата и время последнего изменения |  |

#### UserProfile (Профиль пользователя)
**Назначение:** Расширенные данные профиля игрока (1:1 с User).

| Атрибут      | Описание                         | Ключ |
|-------------|-----------------------------------|------|
| profile_id  | Уникальный идентификатор          | PK   |
| user_id     | Ссылка на пользователя            | FK   |
| avatar_url  | URL аватара                       |      |
| preferred_role | Предпочитаемая роль (attacker/defender) |  |
| created_at  | Дата создания                     |      |
| updated_at  | Дата обновления                   |      |

**Связь:** User 1 — 0..1 UserProfile

| updated_at | Дата обновления                       |      |

#### UserRating (Рейтинг пользователя)
**Назначение:** Рейтинг игрока в рамках конкретного сезона.

| Атрибут    | Описание                          | Ключ |
|-----------|------------------------------------|------|
| rating_id | Уникальный идентификатор          | PK   |
| user_id   | Ссылка на пользователя            | FK   |
| season_id | Ссылка на сезон                   | FK   |

### 5. Запись матча для реплея (детерминистическое воспроизведение)

Реплей строится по принципу: **входы пользователя + случайные исходы → детерминированная симуляция**. Всё остальное (выстрелы, попадания, урон, коллизии) вычисляется физикой при повторном прогоне.

**Карта (maze)** — детерминирована. MazeCreator для уровней 1–3 даёт фиксированную раскладку стен. Сохранять не нужно.

**Сохраняем только:**

1. **Ввод пользователя** — управление (нажатия клавиш)
2. **Спауны танков** — случайные позиции (getRandomSpawnPoint)
3. **Спауны ключей** — случайные позиции при смене уровня
4. **Спауны бонусных ящиков** — случайные момент, позиция и тип (updateBonusBoxSpawning, spawnRandomBox)

---

#### MatchInput (Ввод пользователя)
**Назначение:** Изменения состояния управления (keydown/keyup). Воспроизведение — применение этих событий по tick.

| Атрибут   | Описание                     | Ключ |
|----------|-------------------------------|------|
| input_id | Уникальный идентификатор      | PK   |
| match_id | Ссылка на матч               | FK   |
| tick_number | Тик события               |      |
| player_id | Идентификатор игрока         |      |
| action_type | Действие: forward / backward / turn_left / turn_right / turret_left / turret_right / shoot |  |
| is_pressed | Клавиша нажата (true) или отпущена (false) |  |

**Связь:** Match 1 — 0..* MatchInput

**Примечание:** На каждом тике состояние ввода = совокупность всех MatchInput с tick ≤ текущий. Выстрелы, движение и т.п. выводятся симуляцией.

---

#### MatchTankSpawn (Спаун танка)
**Назначение:** Результат случайного выбора позиции (getRandomSpawnPoint). Используется при старте уровня и респауне.

| Атрибут   | Описание                     | Ключ |
|----------|-------------------------------|------|
| spawn_id | Уникальный идентификатор      | PK   |
| match_id | Ссылка на матч               | FK   |
| tick_number | Тик (0 — старт/смена уровня, иначе — респаун) |  |
| role     | Роль: attacker / defender     |      |
| spawn_line | Строка сетки спауна (0..4) |  |
| spawn_column | Столбец сетки спауна (0..10) |  |
| angle_index | Индекс угла 0–3 (0, π/2, π, 3π/2) — только для респауна |  |

**Связь:** Match 1 — 0..* MatchTankSpawn

**Логика:** При tick=0 и смене уровня — два спауна (attacker, defender). При респауне — один спаун с angle_index. Координаты получаются из spawn_line, spawn_column через PointSpawner.getSpawnPoint.

---

#### MatchKeySpawn (Спаун ключа)
**Назначение:** Случайная позиция ключа при инициализации уровня.

| Атрибут   | Описание                     | Ключ |
|----------|-------------------------------|------|
| spawn_id | Уникальный идентификатор      | PK   |
| match_id | Ссылка на матч               | FK   |
| level    | Уровень (1, 2 или 3)          |      |
| key_index | Номер ключа на уровне (0..REQUIRED_KEYS_PER_LEVEL-1) |  |
| spawn_line | Строка сетки                 |      |
| spawn_column | Столбец сетки              |      |

**Связь:** Match 1 — 0..* MatchKeySpawn

---

#### MatchAmmoBoxSpawn (Спаун бонусного ящика)
**Назначение:** Случайный момент, позиция и тип ящика (spawnRandomBox, getRandomBoxType).

| Атрибут   | Описание                     | Ключ |
|----------|-------------------------------|------|
| spawn_id | Уникальный идентификатор      | PK   |
| match_id | Ссылка на матч               | FK   |
| tick_number | Тик появления ящика        |      |
| spawn_line | Строка сетки                 |      |
| spawn_column | Столбец сетки              |      |
| box_type | Тип: bulLight / bulMedium / bulHeavy / bulGrenade / bulSniper (0–4) |  |

**Связь:** Match 1 — 0..* MatchAmmoBoxSpawn

---

**Воспроизведение реплея:**

1. Инициализировать уровень 1 (карта из MazeCreator, без рандома).
2. Спауны танков и ключей из MatchTankSpawn, MatchKeySpawn.
3. На каждом тике:
   - применить MatchInput (текущее состояние ввода);
   - при tick = запись из MatchAmmoBoxSpawn — создать ящик;
   - запустить симуляцию (физика, коллизии, урон);
   - при респауне танка — взять следующую запись MatchTankSpawn для этой роли.
4. При смене уровня — загрузить MatchTankSpawn и MatchKeySpawn для нового уровня.
5. Конец матча — по условиям checkGameEnd.

**Требования к воспроизведению:** фиксированный timestep (одинаковый deltaTime на тик), сброс ModelIDTracker в начале матча, использование сохранённых случайных исходов вместо Math.random.

**Итог:** Карта не хранится. Выстрелы, попадания, подборы и т.п. не хранятся — восстанавливаются симуляцией.

### 6. Реплеи

#### Replay (Реплей)
**Назначение:** Сохранённая запись матча для повторного просмотра.

| Атрибут          | Описание                      | Ключ |
|-----------------|--------------------------------|------|
| replay_id       | Уникальный идентификатор       | PK   |
| match_id        | Ссылка на матч                 | FK   |
| created_by_user_id | Пользователь, сохранивший запись | FK |
| title           | Название реплея                |      |
| is_public       | Признак публичности            |      |
| created_at      | Дата создания                  |      |
| updated_at      | Дата обновления                |      |

**Связи:**
- Match 1 — 0..* Replay (один матч может быть сохранён как несколько реплеев разными пользователями)
- User 1 — 0..* Replay (пользователь может сохранять множество реплеев)

```

User 0..*───1 UserRating
Season 1────0..* UserRating
Season 0..1─0..* Match
MatchType 1─0..* Match

Match 1────0..* MatchTankSpawn
Match 1────0..* MatchKeySpawn
Match 1────0..* MatchAmmoBoxSpawn
Match 1────0..* Replay
User 1─────0..* Replay
```

## Расширяемость модели

- **Сезоны и рейтинг:** Сущности Season и UserRating позволяют реализовать сезонный лидерборд и архивацию рейтинга.
- **Типы матчей:** MatchType обеспечивает добавление новых режимов (kill-time, кастомные правила).
- **Реплеи:** Детерминистическое воспроизведение по MatchInput + MatchTankSpawn + MatchKeySpawn + MatchAmmoBoxSpawn. Карта — фиксированная, выстрелы и попадания — результат симуляции.
- **Наблюдатели:** Роль `spectator` в MatchParticipant позволяет подключать зрителей к матчу.
- **Статистика:** kills_count, deaths_count, is_winner в MatchParticipant дают основу для аналитики и расширенной статистики.
